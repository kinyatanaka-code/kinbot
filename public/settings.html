      Claude.aiのチャットで会話する（コネクタ）

      全員用・管理者権限 このURLでつなげば、そのまま全社員の実績を横断で見られます
      登録手順
      1. 下のURLを「コピー」する
      2. Claude.aiの設定 →「コネクタ」→「カスタムコネクタを追加」を開く
      3. コピーしたURLをそのまま貼り付けて接続する（ログイン画面は出ません）
      4. 接続後、チャットで「kinbotの今月の実績を教えて」のように話しかける
      https://kinbot-production-225f.up.railway.app/mcp?token=kbt_7bcaf98705c30c3655a8b17dbbd85a0e359aba8b68e3f507

      ⚠️ このURLには全社員分のデータを読める管理者トークンが含まれています。社内の関係者以外には共有しないでください。人数が増える・退職者が出るなどのタイミングで、Railwayの API_TOKENS からこのトークンを見直してください。

      Claude Code などで分析する（APIトークン）

      発行済みのAPIトークンを使うと、Claude Code やターミナルから商談データを取得して分析できます。トークンは Railway の環境変数 API_TOKENS で発行します。よく使うトークンをここに貼っておくと、下のコード例に自動で差し込まれてコピーしやすくなります。

      ⚠️ トークンは全社員の商談データを読める鍵です。共有PCでは保存しないでください。保存先はこのブラウザ内のみ（サーバーには送信しません）。漏れた場合は Railway の API_TOKENS から削除すれば即失効します。

      1. ベースURL
      （このアプリのURL）
      2. 商談一覧を取得
      curl -H "Authorization: Bearer <トークン>" \
  /api/meetings
      3. 1商談の詳細（文字起こし・要約・分析）
      curl -H "Authorization: Bearer <トークン>" \
  /api/meetings/<bot_id>
      4. 新プロセスの抽出データ（案件・イベント）
      curl -H "Authorization: Bearer <トークン>" /api/deals
curl -H "Authorization: Bearer <トークン>" /api/deal-events
      Claude Code への指示例
      「/api/deal-events を Bearer トークン <トークン> で取得して、担当者ごとに『初回商談数・今月判断・来月判断・失注・再商談実施・受注』を集計して」

      ヘッダを付けにくいツールでは ?token=<トークン> をURL末尾に付けても認証できます。

      ChatGPTのGPTから会話する（Custom GPT）

      ChatGPTの「GPTを作成」で、kinbot専用のGPTを一度作っておけば、普段のChatGPT画面からそのGPTを開いて「森田さんの今月の商談状況は？」のように聞くだけで、GPTが自動でkinbotに問い合わせて答えます。※ GPT作成が使える ChatGPT Plus / Team / Enterprise のいずれかが必要です。

      全員用・管理者権限 この設定でGPTを作れば、そのまま全社員の実績を横断で見られます
      作成手順
      1. ChatGPTで「GPTを探す」→「＋作成する」を開き、「構成（Configure）」タブを選ぶ
      2. 下の方の「新しいアクションを作成（Create new action）」を押す
      3. 「認証（Authentication）」で APIキー を選び、種類を Bearer にして、下の②のトークンを貼る
      4. 「スキーマ（Schema）」の「URLからインポート（Import from URL）」に、下の①のURLを貼る（6つの操作が表示されればOK）
      5. 左上に名前・説明を入れて「作成する」→「自分のみ」で保存。GPTに「kinbotの今月の実績を教えて」と話しかける
      ① スキーマのURL（「URLからインポート」に貼る）
      https://kinbot-production-225f.up.railway.app/gpt-actions-openapi.yaml
      ② APIトークン（認証の「APIキー」欄に貼る）
      kbt_7bcaf98705c30c3655a8b17dbbd85a0e359aba8b68e3f507

      ⚠️ このトークンは全社員分のデータを読める管理者トークンです。作ったGPTは必ず「自分のみ」か「リンクを知っている人（社内限定）」で共有し、社外に公開しないでください。人数が増える・退職者が出るなどのタイミングで、Railwayの API_TOKENS からこのトークンを見直してください。

      「URLからインポート」が使えないとき（スキーマを直接貼る）

      GPTの画面に「URLからインポート」が見当たらない、またはうまく読み込めない場合は、下のボタンでスキーマ全文をコピーして、「Schema」欄にそのまま貼り付けてください。

            フェーズ分析の「チーム別」集計に使います。商談の営業担当者名とここの担当者名が一致した分がチームに集計されます。異動などで変わったらここで変更してください。

            カレンダー照合の代表者

            インターン生のカレンダーを読むために使うGoogle連携を選びます。ここで指定した人の連携を経由して照合されるため、他のメンバーはGoogle未連携でも「カレンダーと照合」を実行でき、全員がアポ状況を見られます。候補には、Google連携済みの人だけが表示されます。

            商談予定の自動作成（招待方式）

            アポ振り分けで担当を割り当てると、運用者のカレンダーに商談予定を作成し、担当クローザーをゲストとして招待します。招待方式なので、クローザーのカレンダーへの権限は不要です（招待された本人が時間を変更できるようにしています）。
            重要：この機能を使うには、運用者が 設定→連携→Google連携をやり直して、カレンダーへの書き込み権限を承認する必要があります（読み取りだけの連携では作成できません）。

            インターン生（アポ獲得者）の登録

            アポを獲得したインターン生の名前とGoogleカレンダーのメールアドレスを登録します。ここに登録した人のカレンダーの予定名と、kinbotの商談名を突き合わせて、一致した商談に「アポ獲得者」としてその人の名前を記録します。

            ⚠️ 照合には、各インターン生のGoogleカレンダーがあなた（連携済みアカウント）に共有されている必要があります（共有設定で「予定の詳細をすべて表示」）。社内カレンダーが全社共有になっていれば、そのままでも読めます。

            📊 アポ数の照合と集計（ダッシュボード）は 実績 → インターンアポ から行えます。
