            Claude.aiのチャットで会話する（コネクタ）

            Claude Codeのようなツールを使わず、普段のClaude.aiのチャット画面から直接kinbotのデータを読んで会話できます。「森田さんの今月の商談状況は？」のように聞くだけで、Claudeが自動でkinbotに問い合わせて答えます。

            全員用・管理者権限 このURLでつなげば、そのまま全社員の実績を横断で見られます
            登録手順
            1. 下のURLを「コピー」する
            2. Claude.aiの設定 →「コネクタ」→「カスタムコネクタを追加」を開く
            3. コピーしたURLをそのまま貼り付けて接続する（ログイン画面は出ません）
            4. 接続後、チャットで「kinbotの今月の実績を教えて」のように話しかける
            https://kinbot-production-225f.up.railway.app/mcp?token=kbt_7bcaf98705c30c3655a8b17dbbd85a0e359aba8b68e3f507

            ⚠️ このURLには全社員分のデータを読める管理者トークンが含まれています。社内の関係者以外には共有しないでください。人数が増える・退職者が出るなどのタイミングで、Railwayの API_TOKENS からこのトークンを見直してください。

            Claude Code などで分析する（APIトークン）

            発行済みのAPIトークンを使うと、Claude Code やターミナルから商談データを取得して分析できます。トークンは Railway の環境変数 API_TOKENS で発行します。よく使うトークンをここに貼っておくと、下のコード例に自動で差し込まれてコピーしやすくなります。

            ⚠️ トークンは全社員の商談データを読める鍵です。共有PCでは保存しないでください。保存先はこのブラウザ内のみ（サーバーには送信しません）。漏れた場合は Railway の API_TOKENS から削除すれば即失効します。

            1. ベースURL
            （このアプリのURL）
            2. 商談一覧を取得
            curl -H "Authorization: Bearer <トークン>" \
  /api/meetings
            3. 1商談の詳細（文字起こし・要約・分析）
            curl -H "Authorization: Bearer <トークン>" \
  /api/meetings/<bot_id>
            4. 新プロセスの抽出データ（案件・イベント）
            curl -H "Authorization: Bearer <トークン>" /api/deals
curl -H "Authorization: Bearer <トークン>" /api/deal-events
            Claude Code への指示例
            「/api/deal-events を Bearer トークン <トークン> で取得して、担当者ごとに『初回商談数・今月判断・来月判断・失注・再商談実施・受注』を集計して」

            ヘッダを付けにくいツールでは ?token=<トークン> をURL末尾に付けても認証できます。
